Integration Guide

Public distribution repo for the PaymentGateway SDK. This page covers installation and verification steps for Android (Gradle) and iOS (Swift Package Manager).

Quickstart Copy/paste friendly

Android

// settings.gradle(.kts) or build.gradle(.kts)
repositories {
  maven { url = uri("https://hassanwasfy.github.io/paymentgateway_sdk/") }
  mavenCentral()
  google()
}

dependencies {
  implementation("com.edfapay:paymentgateway:<version>")
}

iOS (SPM)

  1. In Xcode: File → Add Packages…
  2. Add package URL: https://github.com/hassanwasfy/paymentgateway_sdk
  3. Choose the product PaymentGateway
The iOS package uses a binary target. The Package.swift file is updated per release with a new url and checksum.

3) How releases are wired

Each SDK release publishes a zip asset in the DIST repo’s GitHub Releases: 

https://github.com/hassanwasfy/paymentgateway_sdk/releases/download/v<version>/PaymentGateway.xcframework.zip

The Package.swift file is updated with:

  • url → points to the release zip asset
  • checksum → computed using swift package compute-checksum
If you manually edit the zip (even re-zip), the checksum changes and SPM will refuse it. Always upload the exact zip used for checksum calculation.

Troubleshooting Common issues

Troubleshooting

Android: “404” when opening the Maven base URL in a browser

A browser may show 404 on folder URLs because GitHub Pages does not list directories. This does not mean Maven is broken.

  • Test a concrete file URL (e.g. a .pom or maven-metadata.xml).
  • Gradle resolves files directly and does not require directory listing.

Android: dependency not found

  • Confirm the repository is added before mavenCentral()/google() if you use repository modes that restrict resolution.
  • Confirm the coordinate matches published artifactId exactly.
  • If you use Gradle dependency caching, try ./gradlew --refresh-dependencies.

iOS: SPM “checksum mismatch”

  • The checksum must match the exact uploaded zip.
  • Re-zipping changes bytes → checksum changes.
  • Ensure Package.swift points to the correct tag and asset URL.
